# Local Docs Q&A

A small question-answering backend that uses **your own local documents** as the
only source of truth. Drop `.txt` / `.md` files into the `docs/` folder, build a
vector index, and ask questions over HTTP. Retrieval is **hybrid** (semantic +
keyword) with a **cross-encoder re-ranking** step, answers are generated by a
Claude model constrained to the retrieved chunks, and every answer passes a
grounding check before it is returned.

## How it works

```
docs/*.txt|*.md
   │  ingestion — POST /ingest (background task) or python -m app.ingest
   ▼
paragraph-aware chunking (app/chunking.py)
   ▼
local embeddings — sentence-transformers/all-MiniLM-L6-v2 (app/embeddings.py)
   ▼
FAISS cosine index persisted to storage/ + in-memory BM25 index (app/vector_store.py, app/retrieval.py)

POST /ask {"question": ...}
   1. hybrid retrieval: dense FAISS search + sparse BM25 search,
      scores min-max normalized and fused (alpha-weighted)      -> candidate pool
   2. re-ranking: a cross-encoder scores every (question, chunk)
      pair jointly; the top-K in re-ranked order become the LLM
      context                                                   (reject if best relevance too low)
   3. Claude answers using ONLY those chunks                    (reject if model says it can't)
   4. grounding check: answer must be semantically close to at
      least one retrieved chunk                                 (reject if it drifted)
   5. return answer + sources + grounding score
```

Embeddings, BM25, and the re-ranker all run fully locally (no API key needed
for ingestion or search). Only the final answer generation calls the Anthropic
API.

### Why hybrid + re-ranking?

- **Dense search** (embeddings) finds semantically similar text even with no
  word overlap — good for paraphrased questions.
- **Sparse search** (BM25) finds exact rare terms — product codes, error names,
  identifiers — that embeddings often blur.
- The two score lists are min-max normalized to [0, 1] and fused as
  `alpha * dense + (1 - alpha) * sparse` (`HYBRID_ALPHA`, default 0.5).
- The **cross-encoder re-ranker** then reads question and chunk *together*
  (rather than comparing independent vectors), which is a much sharper
  relevance judge. The chunks handed to the LLM are exactly the re-ranker's
  top-K, in re-ranked order, maximizing context precision.

## Setup

Requires Python 3.11+.

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows  (Linux/macOS: source .venv/bin/activate)
pip install -r requirements.txt

copy .env.example .env          # then put your ANTHROPIC_API_KEY in .env
```

The first run downloads the embedding model (~90 MB) and the re-ranker model
(~80 MB) from Hugging Face.

## Usage

1. **Add documents** — put `.txt` or `.md` files in `docs/` (two samples included).

2. **Build the index** (either way):

   ```bash
   python -m app.ingest          # CLI, synchronous
   ```

   or, with the server running, kick it off in the background:

   ```bash
   curl -X POST http://localhost:8000/ingest
   ```

3. **Run the API**

   ```bash
   uvicorn app.main:app --port 8000
   ```

4. **Ask questions**

   ```bash
   curl -s http://localhost:8000/ask \
     -H "Content-Type: application/json" \
     -d "{\"question\": \"How often should the coffee machine be descaled?\"}"
   ```

   ```json
   {
     "question": "How often should the coffee machine be descaled?",
     "answer": "Descale the Aurora X200 every three months, or every 60 days if your water is hard...",
     "sources": [{"source": "aurora-coffee-machine.txt", "chunk_id": 0, "score": 0.9976, "text": "..."}],
     "grounding_score": 0.87
   }
   ```

   `sources[].score` is the re-ranker relevance (0..1); sources are listed in
   re-ranked order — the exact context the model saw.

### Asynchronous ingestion

`POST /ingest` never blocks: it registers a background task and returns
**202 Accepted** immediately. The heavy lifting (reading files, chunking,
embedding, writing the index) runs on a worker thread off the event loop, so
`/ask` keeps serving during re-indexing. When the new index is ready it is
swapped into the running app atomically — requests never see a half-built
index.

```bash
$ curl -s -X POST http://localhost:8000/ingest
{"task_id": "5da545fa744c...", "status": "pending", "status_url": "/ingest/status/5da545fa744c..."}

$ curl -s http://localhost:8000/ingest/status/5da545fa744c...
{"task_id": "5da545fa744c...", "status": "completed", "detail": null, "chunks": 3,
 "created_at": "2026-07-13T07:13:48+00:00", "finished_at": "2026-07-13T07:13:49+00:00"}
```

Task states: `pending` → `processing` → `completed` | `failed` (`detail`
carries the error message on failure). Only one ingestion may run at a time;
a second `POST /ingest` while one is active returns **409 Conflict** with the
running task's id.

## Endpoints

| Method | Path                        | Description                                              |
|--------|-----------------------------|----------------------------------------------------------|
| POST   | `/ask`                      | Answer a question from the indexed documents             |
| POST   | `/ingest`                   | Start background (re-)ingestion — returns 202 + task id  |
| GET    | `/ingest/status/{task_id}`  | Poll an ingestion task: pending/processing/completed/failed |
| GET    | `/health`                   | Index status and configuration                           |

## Errors

The API fails loudly instead of returning made-up answers:

| Status | Endpoint  | When                                                                      |
|--------|-----------|---------------------------------------------------------------------------|
| 400    | `/ask`    | The question is empty or whitespace                                       |
| 404    | `/ask`    | No chunk passes the re-ranker relevance gate, or the model reports the docs don't cover it |
| 422    | `/ask`    | The generated answer failed the grounding check (too far from the sources) |
| 502    | `/ask`    | The Anthropic API call failed (missing key, rate limit, network)          |
| 503    | `/ask`    | The index hasn't been built yet                                           |
| 409    | `/ingest` | An ingestion is already running                                           |
| 404    | `/ingest/status` | Unknown task id                                                     |

## Configuration

Everything is overridable via environment variables (or `.env`) — see
[`.env.example`](.env.example).

| Variable                | Default | What it does                                                        |
|-------------------------|---------|----------------------------------------------------------------------|
| `TOP_K`                 | `4`     | Chunks handed to the LLM after re-ranking                            |
| `RETRIEVAL_CANDIDATES`  | `12`    | Size of the fused candidate pool passed to the re-ranker             |
| `HYBRID_ALPHA`          | `0.5`   | Weight of dense vs sparse in score fusion (1.0 = dense only)         |
| `RERANKER_MODEL`        | `cross-encoder/ms-marco-MiniLM-L-6-v2` | Cross-encoder; swap for e.g. `BAAI/bge-reranker-large` |
| `RERANK_MIN_SCORE`      | `0.001` | Relevance gate on the best chunk (see note below)                    |
| `ANSWER_MIN_SIMILARITY` | `0.45`  | Minimum answer↔chunk cosine for the grounding check                  |

> **Note on `RERANK_MIN_SCORE`:** cross-encoder scores are not calibrated
> probabilities. With the default ms-marco model, off-corpus questions bottom
> out around `2e-5` while answerable ones stay above `2e-3`, so the gate is
> deliberately permissive — the LLM's own "not in context" check and the
> grounding validation catch anything that slips through. Re-measure if you
> swap the re-ranker model.

## Project layout

```
app/
  config.py        # env-driven settings
  chunking.py      # paragraph-aware chunking with overlap
  embeddings.py    # local sentence-transformers wrapper
  vector_store.py  # FAISS index + chunk metadata persistence
  retrieval.py     # BM25 sparse index, hybrid fusion, cross-encoder re-ranker
  ingest.py        # docs -> chunks -> embeddings -> index
  tasks.py         # background ingestion task registry (thread-safe)
  rag.py           # retrieve -> rerank -> generate (Claude) -> validate
  main.py          # FastAPI app
docs/              # your source documents
storage/           # generated index (gitignored)
```
