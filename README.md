# Local Docs Q&A

A small question-answering backend that uses **your own local documents** as the
only source of truth. Drop `.txt` / `.md` files into the `docs/` folder, build a
vector index, and ask questions over HTTP. Answers are generated by a Claude
model constrained to the retrieved chunks, and every answer passes a grounding
check before it is returned.

## How it works

```
docs/*.txt|*.md
   │  ingestion (python -m app.ingest)
   ▼
paragraph-aware chunking (app/chunking.py)
   ▼
local embeddings — sentence-transformers/all-MiniLM-L6-v2 (app/embeddings.py)
   ▼
FAISS cosine-similarity index, persisted to storage/ (app/vector_store.py)

POST /ask {"question": ...}
   1. embed the question, retrieve top-k chunks        (reject if nothing relevant)
   2. Claude answers using ONLY those chunks           (reject if model says it can't)
   3. grounding check: answer must be semantically
      close to at least one retrieved chunk            (reject if it drifted)
   4. return answer + sources + grounding score
```

The embeddings run fully locally (no API key needed for ingestion or search).
Only the final answer generation calls the Anthropic API.

## Setup

Requires Python 3.11+.

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows  (Linux/macOS: source .venv/bin/activate)
pip install -r requirements.txt

copy .env.example .env          # then put your ANTHROPIC_API_KEY in .env
```

The first run downloads the embedding model (~90 MB) from Hugging Face.

## Usage

1. **Add documents** — put `.txt` or `.md` files in `docs/` (two samples included).

2. **Build the index**

   ```bash
   python -m app.ingest
   ```

3. **Run the API**

   ```bash
   uvicorn app.main:app --port 8000
   ```

4. **Ask questions**

   ```bash
   curl -s http://localhost:8000/ask \
     -H "Content-Type: application/json" \
     -d "{\"question\": \"How often should the coffee machine be descaled?\"}"
   ```

   ```json
   {
     "question": "How often should the coffee machine be descaled?",
     "answer": "Descale the Aurora X200 every three months, or every 60 days if your water is hard...",
     "sources": [{"source": "aurora-coffee-machine.txt", "chunk_id": 1, "score": 0.62, "text": "..."}],
     "grounding_score": 0.87
   }
   ```

   After adding or changing files in `docs/`, rebuild without restarting:

   ```bash
   curl -X POST http://localhost:8000/reindex
   ```

## Endpoints

| Method | Path       | Description                                    |
|--------|------------|------------------------------------------------|
| POST   | `/ask`     | Answer a question from the indexed documents   |
| POST   | `/reindex` | Rebuild the index from the `docs/` folder      |
| GET    | `/health`  | Index status and configuration                 |

## Errors

The API fails loudly instead of returning made-up answers:

| Status | When                                                                      |
|--------|---------------------------------------------------------------------------|
| 400    | The question is empty or whitespace                                       |
| 404    | No chunk is similar enough to the question, or the model reports the docs don't cover it |
| 422    | The generated answer failed the grounding check (too far from the sources) |
| 502    | The Anthropic API call failed (missing key, rate limit, network)          |
| 503    | The index hasn't been built yet                                           |

## Configuration

Everything is overridable via environment variables (or `.env`) — see
[`.env.example`](.env.example). The two knobs that matter most:

- `RETRIEVAL_MIN_SCORE` (default `0.25`) — minimum question↔chunk cosine
  similarity for the question to count as answerable at all.
- `ANSWER_MIN_SIMILARITY` (default `0.45`) — minimum answer↔chunk cosine
  similarity for the answer to be considered grounded in the sources.

## Project layout

```
app/
  config.py        # env-driven settings
  chunking.py      # paragraph-aware chunking with overlap
  embeddings.py    # local sentence-transformers wrapper
  vector_store.py  # FAISS index + chunk metadata persistence
  ingest.py        # docs -> chunks -> embeddings -> index
  rag.py           # retrieve -> generate (Claude) -> validate
  main.py          # FastAPI app
docs/              # your source documents
storage/           # generated index (gitignored)
```
